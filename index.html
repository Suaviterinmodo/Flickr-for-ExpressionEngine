Flickr for ExpressionEngine

Integrates Flickr into ExpressionEngine

Download .zip Download .tar.gz View on GitHub

Flickr for EE is an ExpressionEngine 2 add-on that fully integrates Flickr into your website. There are two main features:

The first is a new custom field type that allows you to browse your Flickr photos and select a photo to display in your entry. In addition to the photo you can also display the photo's title, description and a link to the photo's page on Flickr. This makes creating a photoblog extremely simple.

The second part is a set of template tag pairs to display your photostream, favorited photos, Flickr tags and more.

Requirements

Make sure your system meets the minimum requirements:

  • ExpressionEngine 2.02 or later
  • PHP 5
  • A Flickr account (free or pro)

NOTE: Support for Matrix 2 is coming soon!

Installation Instructions

Download Flickr v.2.0.2

  1. Download Flickr v.2.0.2 and unzip the file
  2. Upload the /third_party/dww_flickr folder to your /system/expressionengine/third_party/ directory
  3. In your control panel, click Add-ons » Fieldtypes
  4. Install Flickr
  5. See activation for more info on setup

Activation

To maintain a high level of security Flickr uses an "AuthSub" authentication protocol. This enables the Flickr for EE add-on to access the user's Flickr account without ever handling the users's account login information (i.e. username and password). Unfortuantely, it also makes the activiation process a bit more complex.

The steps below should only take a few minutes to complete:

  1. In your control panel, Add-ons » Fieldtypes
  2. Install Flickr for EE if you haven't already, then click 'Flickr' to view the settings
  3. Open a new browser tab and login to your Flickr account
  4. Once logged in, go to You » Your Apps and click the 'Get Another Key' link on the right side of the page
  5. Click the 'Apply for a non-commercial key' link (the box on the left)
  6. Enter the required info (per the screenshot below), then click 'Submit'
  7. Click the 'Edit auth flow for this app' link (below your Flickr Secret code)
  8. Enter your Callback URL (available on your Flickr settings page from Step #3) into the Callback URL field, then click 'Save Changes'
  9. Click the 'View the API Key for this app' link (right sidebar, located under Admin)
  10. Copy and paste your Flickr Key and Secret values into the fields on your EE Flickr settings page, then click 'Apply' to save the settings
  11. Open the EE Flickr settings again and you'll see a green 'Click to activate Flickr' link, click it
  12. You'll be redirected to Flickr to authorize the EE Flickr add-on, click the 'Ok, I'll authorize it" link
  13. If everything was successful all of the fields will be filled in. If you still only see your API and SECRET fields, along with the activiation link, click the activiation link agian to attempt to connect with Flickr again.
  14. Click 'Apply' to save your settings

Creating a Flickr Field

The EE Flickr Add-on gives you a new 'Flickr' fieldtype which allows you to browse your Flickr photos and select an image for displaying within your content.

Once you've installed and actived Flickr for EE, adding a Flickr field is easy.

  1. In your Control Panel, click Admin » Channel Administration » Custom Fields, then click Add/Edit Custom Fields, and click the 'Create a New Custom Field' button in the top-right
  2. Enter a Field Name, Field Label, and optionally, Field Instructions for your authors
  3. Set the Field Type to 'Flickr'

Save your field and you're done! Any weblogs that use this field's group will now have the Flickr field.

Custom Field Tag

Once you've created a Flickr field and selected an image you're ready to display it within your templates. For the sake of example we'll pretend you named your Flickr field 'flickr_field'.

{flickr_field}

Displays the URL of the photo you selected. This is the complete URL to the photo hosted by Flickr. By default the photo displayed will be the 'medium' size, 500px on the longest side.

To display the actual image you would do something like:

<img src="{flickr_field}" alt="{flickr_field:title}"/>

size = "thumb|square|small|medium|large"

Setting the 'size' parameter returns the URL for the other photo sizes.

Here are the dimenions of each size:

  • square = 75x75 pixels
  • thumb = 100 pixels on the longest side
  • small = 240 pixels on the longest side
  • medium = 500 pixels on the longest side
  • large = 1024 pixels the on longest side (only exists for very large original images)

NOTE: Depending on the original size of the photo, not all sizes are available from Flickr. If you uploaded a low resolution photo its unlikely the 'large' size will be available.

Remember! Flickr Community Guidelines specify that if you post a Flickr photo on an external website, the photo must link back to its photo page.

:title Tag

{flickr_field:title}

Displays the title of the photo from Flickr. If there is no title set in Flickr this will be blank.

:description Tag

{flickr_field:description}

Displays the description of the photo from Flickr. If there is no description set in Flickr this will be blank.

:pagelink Tag

{flickr_field:pagelink}

Displays the URL to the photo's page on Flickr. Useful for linking to the photo's page so visitors can view additional sizes, comments, etc.

:owner Tag

{flickr_field:owner}

Displays the id of the photo's owner.

Photostream Tag Pair

{exp:dww_flickr:photostream}{/exp:dww_flickr:photostream}

Displays your most recent photos in chronological order (newest to oldest). By default it will show 10 photos and the size will be set to 'square.' You can override these with the following parameters:

Tag Cloud Tag Pair

{exp:dww_flickr:tagcloud}{/exp:dww_flickr:tagcloud}

Displays a list of your tags from Flickr. This is designed for use as a tag cloud using the available variables. One of the variables available is the proportional tag size (based on the number of photos tagged with that particular tag). By default the maximum and minimum font sizes are 28px and 11px respectively. You can override these with the following parameters:

font_min = "#"

font_max = "#"

The following single variables are available within your tag pair:

{tag_name}

The name of the tag

{tag_link}

The link to the tag's page on Flickr where all your photos tagged with this tag will display

{tag_size}

The proportional size based on the number of your photos tagged with this tag. If you have lots of photos tagged with this tag, the 'tag-size' will be large. If there are few photos tagged with this tag the 'tag_size' will be smaller. by setting the 'font-min' and 'font-max' parameter you can control the maximum

Example Code

{exp:dww_flickr:photostream size="square"}<a href="{flickr_url}"><img src="{flickr_img}" alt="{flickr_title}"/></a>{/exp:dww_flickr:photostream}

Displays your photostream as a row of square thumbnails which link to the photo's page on Flickr

 

{exp:dww_flickr:favorites size="square"}<a href="{flickr_url}"><img src="{flickr_img}" alt="{flickr_title}"/></a>{/exp:dww_flickr:favorites}

Displays your Flickr favorites as a row of square thumbnails which link to the photo's page on Flickr

 

{exp:dww_flickr:photosets size="square"}<a href="{set_url}"><img src="{set_img}" alt="{set_title}"/></a>{/exp:dww_flickr:photosets}

Displays your Flickr photosets as a row of square thumbnails (using the photoset's cover photo from Flickr) which link to the photoset's page on Flickr

 

{exp:dww_flickr:tagcloud}<a href="{tag_url}" style="font-size:{tag_size}">{tag_name}></a>{/exp:dww_flickr:tagcloud}

Displays a tag cloud where the size of the tag is proportional to the number of photos tagged with that particular tag

 

{exp:imgsizer:size src="{flickr_url}" width="200" alt="{flickr_title}" remote="on"}

Use ImageSizer to dynamically resize the image to any size you want. This will cache the photo on your own server instead of pulling it from Flickr every time its displayed.

NOTE:You'll need to use ImageSizer's 'remote' parameter since we're pulling the image from Flickr.

 

<a href="{flickr_field size="medium"}" title="{flickr_field:description}" class="thickbox"><img src="{flickr_field size="thumb"}" alt="{flickr_field:title}"/></a>

Open a larger view of the photo using Thickbox. The photo's description will be used as the image caption. You would need to include the jQuery and Thickbox code in your header for this to work.

 

With Special Thanks To...

Flickr for EE relies on several superb open-source projects for some of the behind the scenes magic. In no particular order these are:

phpFlickr by Dan Coulter - A PHP wrapper class for the Flickr API. We tried several other PHP wrapper classes and none of them came close to performing as well as phpFlickr.

jScrollPane by Kelvin Luck - A jQuery plugin which provides you with custom scrollbars which work consistently across all modern browsers.

PHP Tag Cloud - A great little script to produce a tag cloud with proportionally sized tags.

FancyBox - a jQuery-based "lightbox" for displaying images and html elements.
